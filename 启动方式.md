# NanoClaw 启动方式

## 首次启动

从零开始搭建 NanoClaw，按以下顺序操作：

### 1. 前置条件

- Node.js 22+
- Docker Desktop（运行中）
- pnpm（`npm install -g pnpm`）

### 2. 安装依赖

```bash
pnpm install
```

### 3. 安装 OneCLI

OneCLI 是凭证管理网关，容器内的 API 请求通过它注入密钥，容器本身永远看不到真实 Key。

```bash
# 安装 OneCLI Gateway（Docker 容器，端口 10254/10255）
curl -fsSL onecli.sh/install | sh

# 安装 OneCLI CLI
curl -fsSL onecli.sh/cli/install | sh

# 确保 CLI 在 PATH 中
export PATH="$HOME/.local/bin:$PATH"
```

### 4. 配置 OneCLI

```bash
# CLI 指向本地 Gateway
onecli config set api-host http://127.0.0.1:10254

# 验证 Gateway 可访问
onecli auth status
```

### 5. 创建 API Key Secret

根据你使用的 API 类型选择对应配置：

**Anthropic 官方 API：**

```bash
onecli secrets create \
  --name "Anthropic API Key" \
  --type anthropic \
  --value "sk-ant-你的key" \
  --host-pattern "api.anthropic.com"
```

**智谱 (GLM) 等兼容 API：**

```bash
# 通过 API 创建（CLI 不支持 injection 参数，需走 HTTP）
curl -s -X POST http://127.0.0.1:10254/api/secrets \
  -H "Authorization: Bearer $(onecli auth api-key | jq -r '.apiKey')" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Zhipu API Key",
    "type": "generic",
    "value": "你的智谱API-Key",
    "hostPattern": "open.bigmodel.cn",
    "injectionConfig": {
      "headerName": "Authorization",
      "valueFormat": "Bearer {value}"
    }
  }'
```

也可以在 OneCLI Dashboard 手动配置：浏览器打开 `http://127.0.0.1:10254` → Secrets → Create。

### 6. 配置 .env

```bash
# OneCLI 连接配置
echo 'ONECLI_URL=http://127.0.0.1:10254' >> .env
echo "ONECLI_API_KEY=$(onecli auth api-key | jq -r '.apiKey')" >> .env

# 如果用智谱等兼容 API，还需要设置 base URL
echo 'ANTHROPIC_BASE_URL=https://open.bigmodel.cn/api/anthropic' >> .env

# 飞书配置（如需飞书频道）
echo 'FEISHU_APP_ID=你的AppID' >> .env
echo 'FEISHU_APP_SECRET=你的AppSecret' >> .env
```

> **注意：** 用智谱等兼容 API 时，`ANTHROPIC_BASE_URL` 必须设置，否则 SDK 默认请求 `api.anthropic.com`，OneCLI 匹配不到 secret 会返回 401。

### 7. 构建容器镜像

```bash
./container/build.sh
```

首次构建较慢（安装 Bun、claude-code、agent-browser 等），后续增量构建很快。

### 8. 启动服务

```bash
pnpm run dev
```

看到以下输出表示启动成功：

```text
[INFO] NanoClaw running
[INFO] Feishu WebSocket connection established
[INFO] OneCLI approval handler started
```

### 9. 初始化第一个 Agent

服务启动后，需要创建 agent group 并绑定频道，否则收到的消息无法路由到容器。有两种方式：

#### 方式一：在 Claude Code 中执行 skill（推荐）

在项目目录下打开 Claude Code（`claude` 命令），然后输入：

```text
/init-first-agent
```

Claude Code 会引导你选择频道、设置 Agent 名称、完成绑定。

> 此方式需要安装 Claude Code CLI（`setup/install-claude.sh`），仅用于初始化阶段，部署后不再需要。

#### 方式二：运行初始化脚本（无需 Claude Code）

```bash
# 示例：绑定飞书频道
pnpm exec tsx scripts/init-first-agent.ts \
  --channel feishu \
  --user-id "feishu:ou_你的用户OpenId" \
  --platform-id "oc_你的聊天ID" \
  --display-name "你的名字" \
  --agent-name "Main"
```

脚本需要服务处于运行状态（通过 CLI socket 通信），参数说明：

| 参数 | 说明 | 示例 |
| ------ | ------ | ------ |
| `--channel` | 频道类型 | `feishu`、`telegram`、`discord` |
| `--user-id` | 用户 ID，格式为 `频道类型:用户标识` | `feishu:ou_xxx` |
| `--platform-id` | 聊天的 platform ID | 飞书群聊 `oc_xxx`，DM 同理 |
| `--display-name` | 显示名称 | `张三` |
| `--agent-name` | Agent 名称，可选，默认 `Main` | `Main` |

> 不推荐手动往数据库插数据，容易漏步骤导致消息路由失败。

## 开发环境

直接运行 TypeScript，无需编译，适合本地调试：

```bash
pnpm run dev
```

后台运行并输出日志：

```bash
nohup pnpm run dev > /tmp/nanoclaw-dev.log 2>&1 &

# 实时查看日志
tail -f /tmp/nanoclaw-dev.log
```

## 生产环境

先编译 TypeScript，再用 Node.js 运行：

```bash
pnpm run build
pnpm start
```

### macOS (launchd)

创建 plist 配置文件：

```bash
cat > ~/Library/LaunchAgents/com.nanoclaw.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.nanoclaw</string>
    <key>WorkingDirectory</key>
    <string>/path/to/nanoclaw</string>
    <key>ProgramArguments</key>
    <array>
        <string>node</string>
        <string>dist/index.js</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/tmp/nanoclaw.log</string>
    <key>StandardErrorPath</key>
    <string>/tmp/nanoclaw.log</string>
</dict>
</plist>
EOF
```

管理命令：

```bash
launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist      # 启动（开机自启 + 崩溃自动重启）
launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist    # 停止
launchctl kickstart -k gui/$(id -u)/com.nanoclaw              # 重启
```

查看日志：

```bash
tail -f /tmp/nanoclaw.log
```

### Linux (systemd)

创建 service 文件：

```bash
sudo cat > /etc/systemd/system/nanoclaw.service << 'EOF'
[Unit]
Description=NanoClaw Assistant
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=你的用户名
WorkingDirectory=/path/to/nanoclaw
ExecStart=/usr/bin/node dist/index.js
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
EOF
```

管理命令：

```bash
sudo systemctl daemon-reload                   # 加载配置
sudo systemctl enable nanoclaw                 # 开机自启
sudo systemctl start nanoclaw                  # 启动
sudo systemctl stop nanoclaw                   # 停止
sudo systemctl restart nanoclaw                # 重启
systemctl status nanoclaw                      # 查看状态
```

查看日志：

```bash
journalctl -u nanoclaw -f                      # 实时日志
journalctl -u nanoclaw --since "1h ago"        # 最近 1 小时
```

## 代码更新后的重启流程

### 只改了 host 端代码（src/ 下的文件）

开发环境：

```bash
kill $(ps aux | grep "tsx src/index" | grep -v grep | awk '{print $2}')
docker stop $(docker ps --filter "name=nanoclaw-v2" -q) 2>/dev/null
nohup pnpm run dev > /tmp/nanoclaw-dev.log 2>&1 &
```

生产环境：

```bash
pnpm run build
# macOS
launchctl kickstart -k gui/$(id -u)/com.nanoclaw
# Linux
sudo systemctl restart nanoclaw
```

### 改了容器内代码（container/agent-runner/src/ 下的文件）

需要额外重建镜像并清缓存：

```bash
./container/build.sh                           # 重建 Docker 镜像
rm -rf data/v2-sessions/*/agent-runner-src     # 清掉各 group 缓存的旧源码
```

然后按上面的流程重启服务。

## 注意事项

- **OneCLI Gateway 必须运行：** NanoClaw 服务依赖 OneCLI Gateway（:10255）为容器注入凭证。如果 Gateway 没启动，容器会报 `401 Unauthorized` 或 `Invalid API key`。检查方式：`docker ps --filter name=onecli`。
- **避免双实例运行：** 同一时间只能用一种方式启动。如果 launchd/systemd 在跑生产版本，不要同时手动跑 `pnpm run dev`，否则两个实例会同时处理消息导致重复回复。
- **检查是否有残留进程：** `ps aux | grep "node.*index" | grep -v grep`
- **查看容器内部日志：** `docker logs <容器名>`（容器退出后日志丢失，需要及时查看）
- **容器名格式：** v2 版本容器名为 `nanoclaw-v2-<folder>-<timestamp>`，如 `nanoclaw-v2-main-1777511300513`
